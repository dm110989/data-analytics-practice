# 🧩 Git — понятная шпаргалка с объяснениями

---

# 📁 1. Работа с репозиторием

## Создать новый репозиторий

```bash
git init
```

📌 Что делает:
- создаёт Git-репозиторий в текущей папке
- появляется скрытая папка `.git`

---

## Скачать репозиторий с GitLab/GitHub

```bash
git clone <url>
```

📌 Что делает:
- скачивает проект
- автоматически связывает его с удалённым репозиторием

---

## Проверить связь с GitLab

```bash
git remote -v
```

📌 Показывает:
- куда отправляются изменения (`push`)
- откуда скачиваются (`fetch`)

---

## Привязать удалённый репозиторий

```bash
git remote add origin <url>
```

📌 Используется если:
- репозиторий уже есть локально
- но ещё не подключён к GitLab

---

# 📄 2. Работа с файлами

---

## Проверить изменения

```bash
git status
```

📌 Показывает:
- какие файлы изменены
- какие добавлены
- какие готовы к коммиту

---

## Добавить файл в staging

### Один файл

```bash
git add my_script.py
```

### Все файлы

```bash
git add .
```

📌 Что делает:
- Git начинает готовить изменения к коммиту

---

## Удалить файл из Git

```bash
git rm <file>
```

📌 Удаляет:
- файл
- и информацию о нём из Git

---

# 💾 3. Коммиты

---

## Создать коммит

```bash
git commit -m "описание изменений"
```

📌 Коммит — это:
- сохранённая версия проекта
- снимок изменений

---

## Посмотреть историю коммитов

```bash
git log --oneline --graph --decorate
```

📌 Показывает:
- историю проекта
- ветки
- merge-коммиты

---

# 🔀 4. Ветки

---

## Посмотреть ветки

```bash
git branch
```

📌 Текущая ветка будет со `*`

---

## Создать новую ветку

```bash
git branch feature
```

📌 Только создаёт ветку, но не переключает.

---

## Переключиться на ветку

```bash
git checkout feature
```

или современный вариант:

```bash
git switch feature
```

---

## Объединить ветки

```bash
git merge feature
```

📌 Вливает изменения:
- из `feature`
- в текущую ветку

---

# ☁️ 5. GitLab / GitHub

---

## Отправить изменения

```bash
git push origin main
```

📌 Отправляет коммиты:
- с компьютера
- на GitLab

---

## Скачать и сразу объединить изменения

```bash
git pull
```

📌 Git:
1. скачивает изменения
2. объединяет их с локальным кодом

---

## Только скачать изменения

```bash
git fetch
```

📌 Без merge.

Полезно:
- посмотреть изменения
- не ломая локальный код

---

# 🧹 6. Отмена изменений

---

## Отменить изменения в файле

```bash
git restore <file>
```

📌 Возвращает файл:
- к последнему коммиту

---

## Откатить последний коммит

### Мягкий откат

```bash
git reset --soft HEAD~1
```

📌 Коммит удалится,
но файлы останутся изменёнными.

---

### Жёсткий откат

```bash
git reset --hard HEAD~1
```

⚠️ Полностью удаляет:
- коммит
- изменения файлов

---

## Безопасный откат

```bash
git revert <commit>
```

📌 Создаёт новый коммит,
который отменяет старый.

---

# 🧠 7. Полезные команды

---

## Посмотреть различия

```bash
git diff
```

📌 Показывает:
- что изменилось в коде

---

## Временно спрятать изменения

```bash
git stash
```

Вернуть обратно:

```bash
git stash pop
```

📌 Полезно если:
- нужно срочно переключить ветку
- а изменения ещё не готовы

---

## Создать тег

```bash
git tag v1.0
```

📌 Обычно используют:
- для релизов
- версий проекта

---

# 🚀 Самый частый рабочий сценарий

## Изменил код → отправил на GitLab

```bash
git status
git add .
git commit -m "fixed parser"
git push
```

---

# 🔀 Правильная работа с ветками

```bash
git checkout my_branch
git add .
git commit -m "new feature"

git checkout master
git pull

git checkout my_branch
git merge master

git push origin my_branch
```

📌 Что происходит:
1. работаешь в своей ветке
2. сохраняешь изменения
3. обновляешь master
4. подтягиваешь свежий master в свою ветку
5. отправляешь ветку на GitLab
